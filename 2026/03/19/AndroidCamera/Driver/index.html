Camera Sensor Driver

Camera Sensor 总体概述

Camera Sensor 类型

CCD(Charge-coupled device 电荷耦合器件)

  1. Full Frame CCD：当光线照射到pixel上时，会将光子转换为电子，然后每次只能迁移一行数据到读出寄存器进行电压转换和AD转换（首先是垂直搬运，然后是水平搬运）
  1. Frame-Transfer CCD：跟Full Frame CCD的区别在于多了移位寄存器，在感光后能将电子先转移到移位寄存器，从而不影响感光pixel继续感光
  1. Interline-Transfer CCD：跟Frame-Transfer CCD的区别在于在每个pixel旁边多了个移位寄存器，只需要一次就能将所有电子从感光pixel上转移下来，而不是一行一行转移

CMOS(Complementary metal-oxide-semiconductor 互补式金属氧化物半导体)

​ 在pixel上就能将电子转化为电压，通过导线直接访问每个pixel上的电压信息，在读出寄存器上就能实现AD转换，效率更高

  1. 前照式：以前受限于工艺，只能将金属电路层放在感光pixel前，这会遮挡部分光线，导致性能下降

  2. 背照式：将感光pixel放在金属电路层之前，能有效避免光线被遮挡

CCD vs CMOS

优点 缺点

RAW10 Transmission

Camera Sensor Clock

VTPixelClock（vt_clk）：Video Timing Clock, From sensor PLL，sensor内部使用，影响帧率和曝光时间

VTPixelClock = Framelengthlines x LinelengthPixelClock x FPS

OutputPixelClock：sensor外部使用，MIPI传输

OutputPixelClock = DataratePerLane x Lane number / BitWidth

MIPI Clock = DataratePerLane / 2

Sensor Resolution Control

Camera Sensor Framerate

FPS(Framerate): Sensor每秒输出多少帧图像

FPS = VTPixelClock/frameLengthLines/lineLengthPixelClock

  • VTPixelClock(vt_clk):Video Timing Clock, From sensor PLL，一般确定了后不会改变

  • frameLengthLines(vts):对应sensor实际寄存器值，height + VBlank，通过该值改变帧率（这里可以理解为一帧有多少行pixel）

  • lineLengthPixelClock(hts):对应sensor实际寄存器值, width + HBlank，一般确定了后不会改变（这里可以理解为一行有多少个pixel）

Camera Sensor Exposure

LineReadoutTime = LinelengthPixelClock / VTPixelClock
LineCount = ExposureTime / LineReadoutTime

  • ExposureTime: Sensor 曝光时间（sensor中没有该寄存器，只能通过写入Linecount来实现改变曝光时间）
  • LineReadoutTime：读一行的时间

LineCount <= FramelengthLines – Offset(4, 6, 8)（曝光时间必须小于帧长，避免同一时刻读取两行数据，一次只读一行）

  • reset time：行复位信号，可以理解为sensor到每一行的第一个pixel，准备开始曝光
  • exposure time：想要配置曝光时间只能通过配置linecount来间接配置
  • linereadout time：读出一行需要的时间
  • frame time：读取一帧图像的时间，fps可以通过 1s/frame time来获得，也可以通过读取一行的时间乘以行数来获得

Camera Sensor CPHY/DPHY

DPHY Lane Module

一条Lane有两个Line，有两种模式且可以复用

Dp-Line：正极通道

Dn-Line：负极通道

DPHY Lane Levels

LP模式：两line相互独立

HS模式：两line互为差分

DPHY Lane States

HS模式下，Dp<Dn时为0，反之为1

LP模式下，不同模式分别可以表示四种状态

DPHY High Speed Transimision

从LP切换到HS，需要按顺序LP-11，LP-01，LP-00的state进行切换，并且进入THS-SETTLE，用于让电流平稳，sensor端发送“00011101”给平台开始高速模式，在数据传输结束时，将最后一位数据置反并持续一段时间，最后再切换到LP-11state回到LP模式

此图为开始信号的示波器图形，clk lane先进入高速模式，然后data lane再进入高速模式

此图为一帧图像的传输示波器图，每传输完一行数据都会重新回到LP模式，等待信号回到HS模式

DPHY Combo Mode

同时支持两个分辨率较小的sensor

Camera Sensor CSID/CSI-2

CSI-2 Layer

Multi-Lane Distribution and Merging

只有当有多条lane存在时才需要进行多条lane的分发和合并

发送端负责将以字节(8bit)为单位的数据流按照一定的规则分发到每一条使能的数据LANE上进行传输

接收端负责将每一条使能的数据LANE上接收到的数据按一定的规则合并成以字节(8bit)为单位的数据流

Low Level Protocol

抽象底层协议模型为一条数据LANE传输的情况，高速传输模式(HS)下以包(Packet)为单位传输数据，每个Packet之间是Low Power State(LP)，其中包又分为长包和短包两种，每个包又是由字节(8bit)为单位构成

长包格式

SoT和EoT之间就是一个包(Packet),其中长包由包头(PH)，数据内容(Data)和包尾(PF)三部分组成，包头又由Data ID，Word Count, ECC这三部分组成，Data ID是一个字节(8bit),其中高位的2个bit(Bit[7:6])用来表示虚拟通道(VC),低位的6个bit(Bit[5:0])用来表示DT（数据类型），比如rgb，yuv或者raw格式。WC(16bit)表示要传输的数据内容(Data)的长度大小，EEC为矫错码，能够矫正一个错误位以及检查出两个错误位，最后以一个16位的校验和作为包尾

短包格式

短包仅包括长包的包头，没有数据部分和包尾,当DT值范围在0x00~0x0F时表示该packet是短包，大于0x0F的都是长包，WC部分的内容由DT决定，比如当DT值为0x00/0x01时，WC可以用来表示frame number，可以用来比较是否前后为同一帧；当DT值为0x02/0x03时，WC可以用来表示line number，可以用来比较是否前后为同一行

Virtual Channel

虚拟通道主要用于在相同的LANE上交替传输不同的图像数据，数据格式可以是相同的，也可以是不同的，常见的应用有Sensor Stagger HDR模式，利用虚拟通道传输长短曝光的图像数据，一条通道总共可以衍生出四个虚拟通道

Camera Sensor Bringup Part4（接口函数详解）

接口配置

高通会通过so库中的这俩个接口来进行曝光的计算和设置，由于每个sensor具体的转换关系不同而需要配置

CalculateExposure 函数

pCalculateExposure函数输入参数: pCalculateExposureData

  • real gain：即total gain = again * dgain * ispgain
  • linecount：有曝光时间转换而来
  • middle/short：HDR的时候多帧融合有用
  • sensorResolutionIndex：即sensor resolution mode，对应各种setting

pCalculateExposure函数输出参数: pExposureInfo

pCalculateExposure函数实现

不同sensor的Analog Gain计算方式不同

用VTS和FPS计算的好处是这两个值是固定的，而HTS不是固定值，通常可以用来改变FPS

FillExposureSettings函数

将数据写入sensor中

pFillExposureSettings函数输入参数:pExposureData，在pRegInfo中存储着sensor的寄存器信息

pFillExposureSettings函数输出参数:pRegSettingsInfo，输出的数量和具体的值

pFillExposureSettings函数实现

从pRegInfo中拿出xml中的值填写

将设置同时生效，避免时序上的问题

Camera Sensor Bringup Part5（KMD DTS 配置）

DTS

DTS是一种用于描述硬件信息的数据结构和语言，它是树状结构的（***指高通芯片平台名称）。

  • ***-camera-sensor-qrd.dtsi
    • camera模组相关硬件描述(sensor,eeprom,actuator,flash,etc)
  • ***-camera.dtsi
    • 平台相关camera硬件描述(ife,ipe,csiphy,csid,cci,etc)

SOC节点（flash，cam-res-mgr）

***-camera-sensor-qrd.dtsi （类似于C语言的头文件）

cam_cci 节点

  1. actuator
  1. sensor
  1. eeprom：直接复制对应sensor的配置信息，除了一些角度和配置引用

Camera Sensor Bringup EEPROM

什么是 OTP

OTP：One Time Programming

主要用于校准不同sensor/模组之间的差异，记录模组基本信息，AF，AWB，LSC，PDAF 等基本信息，Tuning工程师利用这些数据来调试图像效果，保障不同设备出来的图像效果一致性。
